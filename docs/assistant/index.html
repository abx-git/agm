Blueprint Pattern

Architecture documentation in Git — two phases, copy and run.

  1. 1
    Create the documentation folder

    In your application repository.

    mkdir -p docs/architecture
  2. 2
    Copy templates and prompts

    From the pattern repo into your app repo. Fill context/always-on.md. Install prompts/core/system-prompt.md in IDE rules (once).

    mkdir -p docs/architecture scripts prompts
cp -R blueprint-pattern/docs/templates/architecture/* docs/architecture/
cp blueprint-pattern/scripts/bp-workflow.sh scripts/
cp -R blueprint-pattern/prompts/workflows prompts/
chmod +x scripts/bp-workflow.sh
  3. 3
    First session

    Creates blueprint.md (work file) and initial docs from templates.

  4. 4
    Follow-up sessions

    Agent reads blueprint.md, completes the next open phase, updates the graph. Repeat until bootstrap is done.

  5. 5
    Close creation

    Review session (report-only, new chat). Then mark phases complete in blueprint.md.

Pick a mode. Copy checkout → run in terminal. Copy prompt → new agent chat.
