Architecture Graph Method (AGM)

Living architecture docs in your repo — one guided path from install to review.

Start here — you do not need to know AGM yet

Fill in your project once below. This assistant generates everything you need for each step. Always work in a new chat per session (except dialog interviews).

Core files (read once)

Architecture content lives in your chosen template folder (c4-light/, adr-first/, …). Three files orchestrate the graph:

File Role Contains
always-on.md Session context Name, stack, source map — every session
blueprint.md Construction plan Phase → target file, status, WRK, reviews, session log
entry-point.md Graph index (agent) Link map to all docs & sources — traversal start each session; optional overview

blueprint.md = construction plan (agent). entry-point.md = graph index for the agent (links only). Humans read architecture in template sections — not by editing entry-point.

Build — steps 1–4 below. Do them in order the first time. Quickstart.

How you run AGM sessions

Both paths use the same workflows. Pick what fits your IDE setup.
