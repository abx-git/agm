Blueprint Pattern

Configure adoption, copy one prompt, start documenting.

Key files (read once)
always-on.md
Stable application context the agent reads every session: name, purpose, stack, source map, doc paths. Not the architecture itself — facts that rarely change.
blueprint.md
Session backlog and reference state: phase checklist, open work items (WRK), reviews, session log. Tracks progress — not a substitute for arc42 (or your template) chapters.
  1. 1
    Configure and run adoption

    Set parameters below. New chat in your app repo → paste the generated prompt. The agent writes the scaffold, fills always-on.md, creates blueprint.md, and starts the first doc section.

    More parameters
  2. 2
    Follow-up sessions

    Work through open phases in blueprint.md (the session backlog). New chat → copy prompt → repeat until bootstrap phases are done.

  3. 3
    Close creation

    New chat, report-only. Validates blueprint.md against the doc graph, checks links, and recommends where to move insights from follow-up sessions into template sections (references, not duplicates).

Pick a mode. New chat → copy prompt → paste.
