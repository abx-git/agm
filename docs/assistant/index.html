Blueprint Pattern

Architecture documentation in Git — two phases, copy and run.

  1. 1
    Download and unpack

    At the root of your application repository. Creates docs/architecture/, prompts/, scripts/.

    Download kit
    unzip blueprint-pattern-adopt.zip
  2. 2
    Configure

    Edit docs/architecture/context/always-on.md. Copy ide/cursor/*.mdc to .cursor/rules/. See ADOPT.md in the kit.

  3. 3
    First session

    Creates blueprint.md (work file) and initial documentation.

  4. 4
    Follow-up sessions

    Work through blueprint.md. Repeat until bootstrap is complete.

  5. 5
    Close creation

    Review session (new chat, report-only).

Pick a mode. Copy checkout → terminal. Copy prompt → new agent chat.
