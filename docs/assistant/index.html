Blueprint Pattern

Architektur-Dokumentation in dein Git-Repo — mit AI, aber unter deiner Kontrolle

Das Blueprint Pattern ist keine Software, die du installierst. Es ist eine Ordnerstruktur + feste Regeln für deinen AI-Assistenten — damit Architektur-Wissen im gleichen Repository wie dein Code liegt und aktuell bleibt.

1 Verstehen Was & warum → 2 Einrichten ~30 Min, einmalig → 3 Nutzen Agent-Chat starten
Hinweis: Wenn du noch „Blueprint Pattern Assistant“ mit dunkler Workflow-Liste siehst, lade die Seite hart neu (Strg+Shift+R) — die Anleitung wurde komplett neu aufgebaut.
1

Was macht das Pattern — und was nicht?

Das Pattern macht

  • Legt Architektur-Docs unter docs/architecture/ in deinem Repo an
  • Verlinkt Kapitel, Schnittstellen und Code per Markdown
  • Gibt dem AI-Agent klare Aufgaben pro Chat („Workflows“)
  • Hält Fortschritt in blueprint.md fest

Das Pattern ist nicht

  • Kein Cloud-Dienst, kein Login
  • Kein autonomer Architekt — du prüfst alles
  • Kein Ersatz für arc42/C4 — es organisiert sie im Repo
  • Kein RAG oder externes Wiki
2

Einmalig einrichten — in deinem App-Repository

Du arbeitest in zwei Repositories: dem Pattern-Repo (Vorlage, einmal klonen) und deinem App-Repo (dort landen alle Docs). Die Befehle kopierst du nacheinander ins Terminal.

Voraussetzungen: Git · AI-IDE (z. B. Cursor) · ~30 Minuten Zeit

A
Pattern-Repo klonen (Vorlage) 
git clone https://github.com/abx-git/blueprint-pattern.git

Ergebnis: Ordner blueprint-pattern/ neben deinem App-Projekt.

B
In dein App-Repo: Doc-Ordner + Skripte kopieren 
# Im Root DEINES App-Repos ausführen (Pfad zu blueprint-pattern anpassen)

mkdir -p docs/architecture
cp -R ../blueprint-pattern/docs/templates/architecture/* docs/architecture/

mkdir -p scripts prompts
cp ../blueprint-pattern/scripts/bp-workflow.sh scripts/
cp -R ../blueprint-pattern/prompts/workflows prompts/
chmod +x scripts/bp-workflow.sh
C

App-Fakten eintragen

Öffne und bearbeite:

docs/architecture/context/always-on.md

  • Wie heißt die App? Wofür ist sie da?
  • Welcher Tech-Stack? Wo liegt der Quellcode?
  • Welche Services/Systeme gibt es?

Der Agent liest diese Datei automatisch — du musst sie nicht in jeden Chat kopieren.

D
Agent-Regeln in die IDE (einmalig) 
blueprint-pattern/prompts/core/system-prompt.md

Cursor: Inhalt in .cursor/rules/ übernehmen (Vorlage siehe blueprint-pattern/.cursor/rules/).
Andere Tools: In CLAUDE.md oder Copilot-Instructions verlinken.

3

Das hast du jetzt in deinem App-Repo

mein-projekt/
├── src/ …                          ← dein Code (war schon da)
├── docs/architecture/              ← NEU: Architektur-Dokumentation
│   ├── blueprint.md                ← Fortschrittsliste (kein Architektur-Text!)
│   ├── entry-point.md              ← Startpunkt zum Navigieren
│   ├── context/always-on.md        ← deine App-Fakten
│   ├── arc42/ …                    ← Architektur-Kapitel (wird befüllt)
│   └── work/ …                     ← Fragen, Analysen, Reviews
├── prompts/workflows/              ← Aufgaben für den Agent
└── scripts/bp-workflow.sh          ← schaltet die aktuelle Aufgabe um
Fertig eingerichtet? Dann starte jetzt Abschnitt 4 — deine erste Agent-Session. Zur ersten Session ↓
4

Erste Agent-Session — Architektur-Docs anlegen

Der Agent legt blueprint.md an und schreibt die ersten arc42-Kapitel. Du musst nur zwei Dinge tun:

1
Terminal — Aufgabe „bootstrap-init“ aktivieren
./scripts/bp-workflow.sh checkout bootstrap-init
2
AI-IDE — neuen Chat öffnen, optional diesen Text senden:
Lade…
3
Du — Ergebnis kurz prüfen (Agent schreibt Dateien + meldet Status) 
Beispiel am Ende des Chats:

„Ich habe blueprint.md und entry-point.md angelegt.
 Template: arc42. Bitte bestätige: Ist Payment ein externer Service?“

→ Du antwortest im Chat. Dann prüfst du die neuen Dateien in docs/architecture/.

Nächstes Mal: ./scripts/bp-workflow.sh checkout bootstrap-continue — der Agent macht mit den nächsten Kapiteln weiter.
