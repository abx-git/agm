Blueprint Pattern

Workflow Contracts

Jeder Contract definiert, was eine Agent-Session in deinem Projekt tun soll — analog zu den Semantic Contracts, nur für Blueprint-Pattern-Workflows.

  1. Contract unten wählen und Session-Prompt kopieren.
  2. Lokal ./scripts/bp-workflow.sh checkout <id> ausführen (Checkout-Befehl mitkopieren).
  3. Neuen Agent-Chat öffnen, Prompt einfügen — fertig.

Kein Login, kein Backend: statische Seite, alles läuft lokal in IDE und Terminal. Session-Marker: [[ANCHOR:…]] (nicht Semantic-Anchors-Vokabular).

Pflicht-Ausgaben am Session-Ende. Der Agent meldet diese Marker — sie ersetzen keine menschliche Prüfung.

  4. Contract wählen → Session-Prompt kopieren.
  5. ./scripts/bp-workflow.sh checkout … im Pattern- oder App-Repo.
  6. Neuen Agent-Chat öffnen (bei Review-Workflows Pflicht).
