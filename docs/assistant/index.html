Blueprint Pattern

Foundation in your repo → plan areas → improve & sync over time.

Core files (read once)

Architecture content lives in your chosen template folder (c4-light/, adr-first/, …). Three files orchestrate the graph:

File Role Contains
always-on.md Session context Name, stack, source map — every session
blueprint.md Construction plan Phase → target file, status, WRK, reviews, session log
entry-point.md Human entry Overview, navigation, source links — no status column

Blueprint and entry-point link to the same docs: blueprint = plan & progress; entry-point = read & navigate.

Foundation: install the scaffold, choose which parts of your architecture to document (multi-select), then adopt. Improve & sync (next tab) is for deepening content or aligning docs with code later.

  1. 1
    Install — get the scaffold

    Downloads prompts and template into your application repo. No area selection required yet.

    Where to run the script

    Run it in the root of your application repository — the project you document (where your source code and .git live).

    Not in the blueprint-pattern repository. Not from the Assistant URL folder.

    1. cd /path/to/your-app (repo root)
    2. Copy script → save as bp-install-run.sh
    3. chmod +x bp-install-run.sh && ./bp-install-run.sh (macOS/Linux) or follow the Windows note in the script

    After success you should see prompts/ and your documentation root (e.g. docs/architecture/) in that repo.
